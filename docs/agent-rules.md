# Astra Agent Rules

## Never do these
- do not invent new routes not listed in docs/routes.md
- do not add future-scope pages into V1
- do not add new design patterns without updating docs/design-system.md
- do not add backend, database, queue, transcoding, or cloud code unless requested
- do not replace shared components with page-specific copies
- do not introduce untyped data flows
- do not add trendy UI effects that reduce readability

## Always do these
- reuse existing components first
- keep naming consistent
- prefer composition over duplication
- preserve responsive behavior
- preserve dark cinematic design language
- keep accessibility in mind for buttons, focus, labels, and contrast
